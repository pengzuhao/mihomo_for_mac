# 代理组与节点选择

本文说明 MetacubexD 面板 **「代理」** 页中各策略组的含义、如何选择，以及与本机**当前选用**的 Clash 配置（如唯一的 `default-*.yaml` 或其它 `*.yaml`）的对应关系。

面板地址：<http://127.0.0.1:9090/ui/#/proxies>

---

## 1. 两个基础概念

| 概念 | 说明 |
|------|------|
| **节点** | 实际出口。当前配置中有：`VLESS-REALITY`（走代理服务器）、`DIRECT`（直连，不经过代理） |
| **策略组** | 给某一类流量用的「选择器」。在面板里点击卡片，即指定该类流量走组内哪个节点 |

当前运行模式为 **规则（rule）**：流量先匹配当前配置里的 `rules`，再进入对应策略组，**不是**四个组同时生效。

---

## 2. 规则与策略组的对应关系

配置文件中相关规则：

```yaml
rules:
  - GEOIP,CN,全球直连
  - DOMAIN-SUFFIX,google.com,全球代理
  - DOMAIN-SUFFIX,youtube.com,全球代理
  - DOMAIN-SUFFIX,github.com,全球代理
  - MATCH,节点选择
```

含义：

| 流量类型 | 使用的策略组 |
|----------|--------------|
| 目标 IP 在中国大陆 | **全球直连** |
| 域名含 google / youtube / github | **全球代理** |
| 其余所有流量 | **节点选择**（兜底） |

---

## 3. 各板块作用与区别

### 3.1 节点选择（兜底组）

- **作用**：规则未单独指定的流量，全部走此组。
- **常见误区**：选成 `DIRECT` 时，大量国外网站（不在规则列表里的）会直连，可能无法访问。
- **推荐**：`VLESS-REALITY`

### 3.2 全球直连（国内组）

- **作用**：匹配 `GEOIP,CN` 的国内流量。
- **推荐**：`DIRECT`（国内直连更快、更稳）

### 3.3 全球代理（指定国外组）

- **作用**：规则中点名的国外域名（google、youtube、github 等）。
- **推荐**：`VLESS-REALITY`

### 3.4 GLOBAL（全局组）

- **作用**：仅在运行模式为 **全局（global）** 时生效；**所有流量**都看这一组的选项。
- **规则模式下**：平时几乎不影响上网，面板里选什么可以忽略。
- **若切换到全局模式**：应选 `VLESS-REALITY`，否则全部直连。

---

## 4. 面板怎么选

1. 打开 **代理** 页。
2. 在每个策略组内 **点击** 目标节点卡片（高亮/粉色 = 当前选中）。
3. 可点击 **闪电图标** 测速，延迟数字越小通常越快。
4. 选择 **立即生效**，无需重启 mihomo。

---

## 5. 推荐日常配置（规则模式）

| 策略组 | 推荐选择 | 原因 |
|--------|----------|------|
| **节点选择** | `VLESS-REALITY` | 兜底流量走代理，国外站点更稳定 |
| **全球直连** | `DIRECT` | 国内流量直连 |
| **全球代理** | `VLESS-REALITY` | Google / GitHub 等走代理 |
| **GLOBAL** | 规则模式下可忽略 | 仅全局模式时需要设为 `VLESS-REALITY` |

---

## 6. 流量走向示意

```
访问目标
   │
   ├─ 国内 IP (GEOIP CN)           → 全球直连 → DIRECT
   ├─ google / youtube / github    → 全球代理 → VLESS-REALITY
   └─ 其它（规则未命中）            → 节点选择 → 你在面板里选定的节点
```

---

## 7. 与系统代理、TUN 的区别

| 功能 | 在哪里控制 | 作用 |
|------|------------|------|
| **策略组 / 节点** | 面板 → 代理 | 决定流量走代理还是直连、走哪个节点 |
| **TUN** | 面板 → 配置 | 虚拟网卡接管系统流量（当前 Desktop 路径下可能无法开启） |
| **系统代理** | macOS 系统设置 或 终端脚本 | 是否把应用流量交给 `127.0.0.1:7890` |

三者相互独立：

- **系统代理开 + 节点选择正确** → 多数应用可正常翻墙（当前常用方案）。
- **TUN 开** → 一般可关系统代理，由 TUN 统一接管。
- **代理页只改节点** → 不会自动开关系统代理或 TUN。

---

## 8. 扩展规则

若某网站打不开，可在当前 yaml 的 `rules` 中增加域名规则，例如：

```yaml
  - DOMAIN-SUFFIX,example.com,全球代理
```

修改后：`~/Desktop/mihomo/startup-mihomo.sh restart`

或在面板 **配置** 页确认运行模式仍为 **规则**，避免误切到 **全局** 且 GLOBAL 组选成了 `DIRECT`。
