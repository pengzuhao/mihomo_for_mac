# Mihomo 服务（Hammerspoon）

类似 ClashX / Stash：**快捷键 + 菜单栏图标**，数据在 `~/.config/mihomo/`。

## 快捷键

| 快捷键 | 作用 | 范围 |
|--------|------|------|
| **⌃⌥⌘ M** | 开：显示菜单栏图标并启动 mihomo（TUN）；关：停止 mihomo 并隐藏图标 | **全局** |
| **⌃⌥⌘ L** | 在 `~/.config/mihomo/` 下**所有** `*.yaml` 间按名称排序循环切换（**仅运行中**；不含启停） | **全局**（运行中且 ≥2 个 yaml 才注册） |
| **单击菜单栏图标** | 打开菜单 → **每个 `*.yaml` 一行可选** + **关闭** + **退出** |

### 菜单说明

- **`~/.config/mihomo/` 下所有 `*.yaml`**（按文件名排序）各占一行；点击即切换当前配置。
- **核心在跑**：API 热切换该 yaml 并尽量开 TUN（与 `mihomo-control.sh set-config` 一致）。热切换失败时才重启核心；**配置切换触发的重启不会弹出「mihomo 已停止」**（手动 `off` / 菜单「关闭」仍会提示）。
- **核心未跑（菜单）**：先 `set-config` 再自动 `on` 拉起，无需再按 **⌃⌥⌘ M**。
- **核心未跑（命令行 `set-config`）**：仍只写入 `.active-config`，需再 `on` 或 **⌃⌥⌘ M**。
- **✓**：运行中表示**当前正在用的** yaml；已关闭时**仅「关闭」**打勾（各 yaml 行不再同时勾选）。
- **secret 不一致**：若当前跑的是带 `secret` 的配置（如 `default-ss-s`），热切换时会用**运行中进程**的 secret 调 API，避免误触发整进程重启。
- **默认**：在 `~/.config/mihomo/` 下须有且仅有 **1 个 `default-*.yaml`**（文件名以 **`default-` 开头**、以 **`.yaml` 结尾**）。若为 **0** 或多个，相关命令会报错到 **stderr**，并打印如何修复（删到仅剩一个 `default-*`，或使用 `.active-config`）；Hammerspoon 调 `mihomo-status.sh` 若失败可能临时显示为「关」。
- **其它**：任意 **`.yaml` 结尾**的文件都会出现在菜单/`cfg_list`，作为可选订阅配置。

**关闭** / **退出** 与其它说明不变；悬停图标可看当前模式与配置名。

> **说明**：曾尝试 **⌘ E** 在图标显示时切换开/关，但 Hammerspoon 的 `hs.hotkey` 为全局注册，会被 Cursor / 浏览器等应用的 ⌘E 抢占，且无法像菜单点击那样可靠刷新图标，故已移除。

## 安装

```bash
~/Desktop/mihomo/install-hotkeys.sh
~/Desktop/mihomo/install-sudoers.sh   # 可选：关闭时不弹管理员密码
```

**系统设置 → 隐私与安全性 → 辅助功能** → 允许 **Hammerspoon**。

`install-hotkeys.sh` 只做：**链 `init.lua`、赋权脚本、重启 Hammerspoon**；**默认不启动** mihomo 核心。**启停交给 ⌃⌥⌘ M**（或 `mihomo-control.sh on|off`）。若仍希望装好就拉起进程（旧行为）：**`MIHOMO_AUTO_START=1 ./install-hotkeys.sh`**。装完后若在终端关掉：**`~/Desktop/mihomo/mihomo-control.sh off`**。

不要用 **`pkill -f mihomo`**：会匹配命令行里任意含 `mihomo` 的进程（例如 **Cursor** 插件进程名里的工作区名），容易误杀。停服务请用 **`~/Desktop/mihomo/mihomo-control.sh off`**。

本仓库里也**不要用**单独的 **`pgrep -f ~/.config/mihomo/mihomo`** 当「是否在跑」：命令行参数里带过该路径（例如 **`zsh -c`** 里的诊断脚本、IDE 封装）会与**真内核**混淆，曾有「误判已运行 → `startup-mihomo` 跳过启动 → 有图标无端口」。现统一改用 **`is_mihomo_core_running`**（**`ps axww` 不截断**，且要求与启动脚本一致的 **` -f `** + **` -d`** 指向 **`DATA_DIR`**）。

**注意：** 链接目标为 **`~/Desktop/mihomo/hammerspoon-init.lua`** 的**绝对路径**。若整套项目移到别的目录，需重新做一次链接：

```bash
ln -sf "<新项目路径>/hammerspoon-init.lua" ~/.hammerspoon/init.lua
```

或先改 `install-hotkeys.sh`（及仓库内其它脚本）里的 **`SCRIPT_DIR`** 指向新路径，再执行 `install-hotkeys.sh`。

## 命令行

```bash
~/Desktop/mihomo/mihomo-control.sh on
~/Desktop/mihomo/mihomo-control.sh off
~/Desktop/mihomo/mihomo-control.sh switch
~/Desktop/mihomo/mihomo-control.sh switch-config
~/Desktop/mihomo/mihomo-control.sh set-config default-ss-s.yaml   # 或任意 *.yaml 基名；核心未跑时仅写 .active-config
```

### 多配置

`~/.config/mihomo/` 下**任意** `.yaml`（扩展名必须为 **`.yaml`**）都会列入菜单与 `cfg_list`。

- **谁当默认**：无有效 `.active-config` 时，仅当 **`default-*.yaml` 恰好匹配 1 个文件** 才用之；若无匹配或多个匹配 → 终端/Hammerspoon 会看到说明，请删到只剩一个 `default-*`，或：`echo \"你的.yaml\" > ~/.config/mihomo/.active-config`。
- **⌃⌥⌘ L**：运行中且目录下 **≥2 个** `*.yaml` 时循环切换。

面板：<http://127.0.0.1:9090/ui/>

监听端口是否起来（`netstat | grep 7890` 有时不可靠）：

```bash
lsof -nP -iTCP:7890 -sTCP:LISTEN
lsof -nP -iTCP:9090 -sTCP:LISTEN
```

不要用 **`netstat | grep 7890`** 当「是否在听」的唯一依据：常会看到 **`TIME_WAIT`**，那是**已结束连接**的善后状态，**不是** `LISTEN`；核心若已退出，`TIME_WAIT` 过一会儿也会全部消失，看起来像「突然没 7890」。

**判断是否真在提供服务**（三选一）：

```bash
pgrep -fl "${HOME}/.config/mihomo/mihomo"
lsof -nP -iTCP:7890 -sTCP:LISTEN
curl -sS -m 2 -o /dev/null -w '%{http_code}\n' http://127.0.0.1:9090/   # 有核心时多为 401/404 等，超时则多半没起来
```

**上不了网**且 **7890 没在听**：多半 **系统代理仍指向 127.0.0.1:7890**，先救急：

```bash
~/Desktop/mihomo/mihomo-control.sh clear-proxy
```

再通过菜单 / 快捷键正常开一次。

## 排错

若 Hammerspoon Console 仍有红字，查看日志：

```bash
tail -f ~/.config/mihomo/hammerspoon.log
```

正常加载应出现一行 `OK menubar+hotkeys`。

- 若 **`mihomo.log`** 反复出现 **`Mihomo shutting down`** 与 **`listening at: ...7890`** 交替：多见为 **关/开并发**（旧版用 `stopServiceAsync`）。**务必 Hammerspoon → Reload Config**，使 `~/.hammerspoon/init.lua`（若已 `ln -s` 到本仓库）加载到 **走 `runControl off` 串行 busy** 的版本；Reload 后日志里应 **不再出现** `stopServiceAsync done`。

## 说明

- **⌃⌥⌘ M**（再次）/ **退出**：`delete()` 彻底移除图标（无空位）；再开时用固定 `autosaveName` 尽量恢复原位置。
- **TUN** / **关闭**：点菜单即可。**开/关会通过 `hs.execute` 同步跑 `mihomo-control.sh`**（与终端一致）。执行时 **Hammerspoon 可能卡住数秒**。状态以 **`mihomo-status.sh` + `pgrep`** 为准；若核心已启动但 **`~/.config/mihomo/.switch-on` 仍为 off**（曾因纠偏写入），仍会显示为「开着」（脚本已兜底）。菜单约 **8 秒**随定时刷新；也可 **⌃⌥⌘ M** 关再开一次强制刷新。
- 关闭服务时若已装 `install-sudoers.sh`，不再询问管理员密码。
- 源图：`ico-openup.png`（开）、`ico-closed.png`（关）；菜单栏用 `menu-icon-on.png` / `menu-icon-off.png`（18px，含 @2x）
